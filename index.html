Title: [Register][API][Validation] — API allows successful registration with invalid password

Environment: Web App | API Endpoint: /api/register | Build 2.1.3

Preconditions: API endpoint is accessible; Postman or similar tool is configured.

Steps to Reproduce:

  • Check the brand's password policy: passwords must contain both letters and numbers.
  • Open Postman (or similar API testing tool).
  • Send a registration request with an invalid password (e.g., \"123456\").
  • Observe the response.

Expected Result:

  • The API should return a 400 Bad Request or equivalent error indicating password is invalid.

Actual Result:

  • The API returns a 200 OK status and successfully registers the user, bypassing password validation.

Impact: Security compliance is broken. Password policy enforcement is bypassed, allowing users to register with weak or easily guessable passwords.

Severity: Critical

Priority: High — Compromises user account security and regulatory requirements.
